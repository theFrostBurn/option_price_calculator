# AGENTS.md 가이드 문서

## AGENTS.md란?

AGENTS.md는 AI 코딩 에이전트를 위한 간단하고 개방적인 형식의 지침 파일입니다. 현재 6만 개 이상의 오픈소스 프로젝트에서 사용되고 있습니다.

**AGENTS.md는 에이전트를 위한 README**라고 생각하시면 됩니다. AI 코딩 에이전트가 프로젝트 작업을 수행하는 데 필요한 컨텍스트와 지침을 제공하는 명확하고 예측 가능한 장소입니다.

## 왜 AGENTS.md가 필요한가?

- **README.md**는 사람을 위한 것: 빠른 시작, 프로젝트 설명, 기여 가이드라인
- **AGENTS.md**는 에이전트를 위한 것: 빌드 단계, 테스트, 규칙 등 에이전트가 필요로 하는 상세한 컨텍스트

README를 간결하게 유지하면서 에이전트에게 정확한 지침을 제공하기 위해 의도적으로 분리했습니다.

## 주요 특징

### 1. 범용성
하나의 AGENTS.md 파일이 여러 AI 코딩 에이전트와 호환됩니다.

### 2. 유연성
필수 필드가 없습니다. 표준 마크다운으로 원하는 제목과 내용을 자유롭게 작성할 수 있습니다.

### 3. 우선순위 체계
- 편집 중인 파일에 가장 가까운 AGENTS.md가 우선순위를 가집니다
- 사용자의 명시적인 채팅 프롬프트가 모든 것을 오버라이드합니다

## AGENTS.md 작성 방법

### 1단계: AGENTS.md 파일 생성
레포지토리 루트에 AGENTS.md 파일을 생성합니다. 대부분의 코딩 에이전트에게 요청하면 자동으로 생성해줄 수도 있습니다.

### 2단계: 중요한 내용 작성
에이전트가 프로젝트를 효과적으로 작업하는 데 도움이 되는 섹션을 추가합니다.

**인기 있는 섹션들:**
- 프로젝트 개요
- 빌드 및 테스트 명령어
- 코드 스타일 가이드라인
- 테스트 지침
- 보안 고려사항

### 3단계: 추가 지침 작성
- 커밋 메시지 또는 풀 리퀘스트 가이드라인
- 보안 관련 주의사항
- 대용량 데이터셋
- 배포 단계
- 새로운 팀원에게 알려줄 모든 것

### 4단계: 모노레포의 경우 중첩 AGENTS.md 사용
각 패키지 내부에 별도의 AGENTS.md를 배치할 수 있습니다. 에이전트는 디렉토리 트리에서 가장 가까운 파일을 자동으로 읽으므로, 각 서브프로젝트가 맞춤형 지침을 제공할 수 있습니다.

## 기본 예시

```markdown
# AGENTS.md

## 설정 명령어
- 의존성 설치: `pnpm install`
- 개발 서버 시작: `pnpm dev`
- 테스트 실행: `pnpm test`

## 코드 스타일
- TypeScript strict 모드
- 싱글 쿼트, 세미콜론 미사용
- 가능한 함수형 패턴 사용
```

## 상세 예시

```markdown
# 샘플 AGENTS.md 파일

## 개발 환경 팁
- `ls`로 스캔하는 대신 `pnpm dlx turbo run where <project_name>`을 사용하여 패키지로 이동하세요.
- `pnpm install --filter <project_name>`을 실행하여 작업 공간에 패키지를 추가하면 Vite, ESLint, TypeScript가 인식할 수 있습니다.
- `pnpm create vite@latest <project_name> -- --template react-ts`를 사용하여 TypeScript 체크가 준비된 새로운 React + Vite 패키지를 생성하세요.
- 각 패키지의 package.json 내부의 name 필드를 확인하여 올바른 이름을 확인하세요. 최상위 레벨은 건너뛰세요.

## 테스트 지침
- .github/workflows 폴더에서 CI 계획을 찾으세요.
- `pnpm turbo run test --filter <project_name>`을 실행하여 해당 패키지에 정의된 모든 체크를 실행하세요.
- 패키지 루트에서는 `pnpm test`만 호출하면 됩니다. 병합하기 전에 모든 테스트를 통과해야 합니다.
- 한 단계에 집중하려면 Vitest 패턴을 추가하세요: `pnpm vitest run -t "<test name>"`
- 전체 테스트 스위트가 녹색이 될 때까지 테스트 또는 타입 오류를 수정하세요.
- 파일을 이동하거나 import를 변경한 후 `pnpm lint --filter <project_name>`을 실행하여 ESLint 및 TypeScript 규칙이 여전히 통과하는지 확인하세요.
- 아무도 요청하지 않아도 변경한 코드에 대한 테스트를 추가하거나 업데이트하세요.

## PR 지침
- 제목 형식: [<project_name>] <Title>
- 커밋하기 전에 항상 `pnpm lint`와 `pnpm test`를 실행하세요.
```

## 자주 묻는 질문

### Q: 필수 필드가 있나요?
A: 아니요. AGENTS.md는 표준 마크다운입니다. 원하는 제목을 사용하면 되며, 에이전트가 제공한 텍스트를 파싱합니다.

### Q: 지침이 충돌하면 어떻게 되나요?
A: 편집 중인 파일에 가장 가까운 AGENTS.md가 우선이며, 명시적인 사용자 채팅 프롬프트가 모든 것을 오버라이드합니다.

### Q: 에이전트가 AGENTS.md의 테스트 명령어를 자동으로 실행하나요?
A: 네, 목록에 포함시키면 실행합니다. 에이전트는 관련된 프로그래밍 체크를 실행하고 작업을 완료하기 전에 실패를 수정합니다.

### Q: 나중에 업데이트할 수 있나요?
A: 물론입니다. AGENTS.md를 살아있는 문서로 취급하세요.

### Q: 기존 문서를 AGENTS.md로 마이그레이션하려면?
A: 기존 파일 이름을 AGENTS.md로 변경하고 하위 호환성을 위해 심볼릭 링크를 생성하세요:

```bash
mv AGENT.md AGENTS.md && ln -s AGENTS.md AGENT.md
```
